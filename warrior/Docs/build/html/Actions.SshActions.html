open_ssh_session(system_name, session_name=None)[source]¶

Connects to the ssh port of the the given system and creates a paramiko session object for the system

A session_id is created using the combintation system_name+session_name and returned by this keyword to be stored in Warrior framework’s data repository

Tags to be used in input datafile to retrieve values for open_ssh_session <ip> = IP address of the system <username> = login username of the system <password> = login password of the system

Arguments:
  1. system_name(string) = Name of the system from the input datafile
  2. session_name(string) = name of the session to the system
Returns:
  1. status(bool)= True / False
  2. session_id (dict element)= key, value
send_ssh_cmds_by_testdata(system_name, session_name=None)[source]¶

Sends all commands from all rows that are marked execute=yes from the testdata - This keyword expects the usage of warrior framework’s

recommended testcase datafile and testdata xml files.
  • Warrior framework’s recommended sample testcase datafile is
    available in Warriorspace/Data/sample_Data.xml
  • This keyword expects the testcase datafile to have a testdata tag pointing to

the location of the testdatafile. - The Description and instructions on how to use the recommended testdata file is available

in the sample testdata file located at Warriorspace/Config_files/sample_testdata.xml
  • This keyword expects a paramiko session with the combination of
    system_name and session to be available in the Warrior Framework’s data repository. i.e. the session should have been established using open_ssh_session method in this file.
Arguments:
  1. system_name (string) = name of the system in the input datafile
  2. session_name(string) = name of the session to the string
Returns:
  1. status (bool)
